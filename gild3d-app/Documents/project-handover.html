05   Known Issues & Limitations
IssueSeverityDescriptionRecommendation
No password reset flow Medium Members who forget their password have no self-service recovery. Admin must manually assist. Implement email-based password reset link in Phase 2. Priority before large member base.
No booking email notifications Medium Members do not receive email confirmations for bookings or payment confirmations. Add transactional email triggers for booking state changes in Phase 2.
Companion profiles are admin-only Low (by design) Companions cannot self-manage their profiles. All edits require admin intervention. Phase 2: companion self-service portal with admin approval workflow.
PostgreSQL backups not automated High — Action Required Database data is in a Docker volume with no automated backup. A server failure could cause data loss. Configure immediate: AWS Lightsail instance snapshots + PostgreSQL dump via cron to S3.
Single-region deployment Medium Platform runs on a single AWS Lightsail instance with no geographic redundancy. Acceptable for Phase 1. At 500+ active members, migrate to ECS + RDS Multi-AZ.
No native mobile app Low Web-only experience. Mobile-responsive but not native app. Phase 3: React Native apps targeting iOS and Android.
Messaging is not real-time Medium Messages are stored and retrievable but do not push in real-time. Users must refresh to see new messages. Phase 2: WebSocket integration (Socket.io) for real-time messaging.
No GDPR data deletion portal Medium Data subject access requests and deletion must be handled manually by admin. Required for EU/UK compliance. Phase 2: self-service data export and deletion. Required before operating in EU/UK markets at scale.

IMMEDIATE ACTION REQUIRED: PostgreSQL automated backups must be configured before any live member transactions occur. A database failure without backups represents an unrecoverable data loss scenario. See the Technical Architecture document Operational Runbook for backup configuration commands.

06   Recommended Phase 2 Scope

The following items are recommended for Phase 2 development, prioritised by business impact:

P1 — Critical

Database Automated Backups

Immediately required before production transactions. Daily PostgreSQL dumps to S3 with 30-day retention.

P1 — High

Password Reset Flow

Email-based self-service password reset. Required for member retention and support load reduction.

P1 — High

Booking Email Notifications

Transactional emails for booking confirmation, payment receipt, cancellation. Significantly improves member trust.

P2 — Medium

Real-Time Messaging (WebSocket)

Socket.io integration for live member-companion messaging. Key differentiator for premium experience.

P2 — Medium

Companion Availability Calendar

Prevent double-booking and improve booking conversion by showing companion availability slots.

P2 — Medium

Companion Self-Service Portal

Allow companions to manage their own profiles (with admin approval). Reduces admin operations load.

P2 — Medium

GDPR Data Rights Portal

Self-service data export and deletion requests. Required for EU/UK market compliance at scale.

P3 — Future

Native Mobile Applications

iOS + Android apps via React Native. Increases engagement and enables push notifications.

P3 — Future

Review & Rating System

Moderated member reviews of companions. Builds trust, surfaces quality and increases conversion.

P3 — Future

Multi-Region Infrastructure

Migrate from Lightsail to ECS + RDS Multi-AZ for high availability at scale (>500 concurrent users).
