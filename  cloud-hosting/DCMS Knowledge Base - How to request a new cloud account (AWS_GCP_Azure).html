How to request a new cloud account (AWS/GCP/Azure)

1.   Summary

This SOP provides details on how to request a new AWS/GCP/Azure cloud account via the internal portal

2.   Audience / Prerequisites

  • Cloud Architects
  • Must be an Okta user and have ServiceNow ITIL user access
  • Must have the DCMS CMS Portal Role and be added to the Deloitte Cloud Managed Service ‘company’ to enable redirect
  • Ensure you are in the correct domain MSP/DCMS/CMS

3.   Procedure

3.1.  Process steps

3.1.1 Request a new AWS account

1. Navigate to the Portal via ServiceNow or with this link



2. Select Request Something




3. Select OpenCloud On-Boarding then Request a New Cloud Account



4. The account request form will open:



5. The form is divided into three main sections. Fill out each section of the form with as much information as possible:

** denotes mandatory field

  • Requested for: this section is pre-populated with your name but this can be changed by selecting the box and choosing from the drop-down menu if needed **




  • Project ID: select the appropriate project from the drop-down menu **

  • Project Name: this section will be populated once the Project ID above has been selected

  • Client Name: this section will be populated once the Project ID above has been selected

  • Cloud Provider: select AWS from the drop-down menu **

  • PPA (Preferred Pricing Agreement) account request: select Yes or No. For further explanation on this, please select the ? icon (if further clarification is required, reach out to the Digital Platform team for assistance) **

  • Request Type: select either Client (single account for an existing project) or Stub and Client (creates both a Stub AND Client account for new projects) from the drop-down menu. Note that selecting Stub will add an additional mandatory section include both Stub and Client account specifications  **


 

  • Environment: select Production or Dev from the drop-down menu **

  • Primary Region: select the appropriate region from the drop-down menu **

  • Network Configuration: select the appropriate network config from the drop-down menu **

  • Networking Address: enter the networking address **

  • OpenVPN Access Server required for Client Account?: select either Yes or No from the drop-down menu **

  • Account Description: enter as much information as possible here **

  • Integrated Tooling: select Yes/No from the drop-down menu for Prisma and Datadog (Prisma is not available for PPA accounts as they will be automatically onboarded to the GCSS provided compliance solution). Provide explanation if tools are unselected. Trend and Tenable are onboarded to all new accounts (AWS/GCP/Azure) by default and these fields are read-only (If PPA is selected, the account will be automatically onboarded to Qualys, rather than Prisma)

 

Tip: Selecting No on any of the tools will reveal an explanation box for reasoning - this is also mandatory




  • Data Classification: select Confidential, High-risk Confidential, Protected Health Confidential or Public from the drop-down menu **

  • Data Description: enter details of the potential data the account will hold **

  • Supported by: select the pod that will support the account from the drop-down menu **

  • Additional information: if there is any additional information that will assist in the account creation, enter it here 

  • Date required: enter the date by which the account is needed - note that there is currently a 3 day SLA on account requests **

  • Data Ownership: select Client or Deloitte from the drop-down menu **

6. Finally, select Submit. The new AWS cloud account has now been requested and account creation will be actioned by a member of the Products Engineering team


3.1.2 Request a new GCP account

1. Navigate to the Portal via ServiceNow or with this link



2. Select Request Something



3. Select OpenCloud On-Boarding then Request a New Cloud Account

4. The account request form will open:



5. The form is divided into three main sections. Fill out each section of the form with as much information as possible:

** denotes mandatory field

  • Requested for: this section is pre-populated with your name but this can be changed by selecting the box and choosing from the drop-down menu if needed **



  • Project ID: select the appropriate project from the drop-down menu **

  • Project Name: this section will be populated once the Project ID above has been selected

  • Client Name: this section will be populated once the Project ID above has been selected

  • Cloud Provider: select GCP from the drop-down menu **

  • Request Type: select either Client (single account for an existing project) or Stub and Client (creates both a Stub AND Client account for new projects) from the drop-down menu. Note that selecting Stub will add an additional mandatory section include both Stub and Client account specifications **

  • Environment: select Production or Dev from the drop-down menu **

  • Primary Region: select the appropriate region from the drop-down menu **

  • Network Configuration: select the appropriate network config from the drop-down menu **

  • Networking Address: enter the networking address **

  • OpenVPN Access Server required for Client Account?: select either Yes or No from the drop-down menu **

  • Account Description: enter as much information as possible here **

  • Integrated Tooling: select Yes/No from the drop-down menu for Datadog. Provide explanation if unselected. Trend and Tenable are onboarded to all new accounts (AWS/GCP/Azure) by default and these fields are read-only

 

Tip: Selecting No on any of the tools will reveal an explanation box for reasoning - this is also mandatory

  • Data Classification: select Confidential, High-risk Confidential, Protected Health Confidential or Public from the drop-down menu **

  • Data Description: enter details of the potential data the account will hold **

  • Supported by: select the pod that will support the account from the drop-down menu **

  • Additional information: if there is any additional information that will assist in the account creation, enter it here 

  • Date required for?: enter the date by which the account is needed **

  • Data Ownership: select Client or Deloitte from the drop-down menu **

6. Finally, select Submit. The new GCP cloud account has now been requested and account creation will be actioned by a member of the Products Engineering team

3.1.3 Request a new Azure account

1. Navigate to the Portal via ServiceNow or with this link



2. Select Request Something



3. Select OpenCloud On-Boarding then Request a New Cloud Account

4. The account request form will open:



5. The form is divided into three main sections. As above, fill out each section of the form with as much information as possible (if unsure, please contact the Product Engineering team)

6. Finally, select Submit. The new Azure cloud account has now been requested and account creation will be actioned by a member of the Products Engineering team
