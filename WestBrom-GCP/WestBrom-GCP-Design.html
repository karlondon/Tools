1. Solution Overview

This solution deploys the GCP-side infrastructure for the WestBrom (Converge Banking) cross-cloud off-site backup strategy. Aurora PostgreSQL backups and Cognito backups are transferred nightly from AWS S3 to GCP Cloud Storage via the Storage Transfer Service, providing cloud-level isolation in the event of a widespread AWS regional outage.

9. Post-Deployment Checklist

Complete each item after a successful Terraform apply to validate the end-to-end solution.

Infrastructure Validation
  • Verify GCP APIs are all enabled (7 services)
  • Confirm KMS key ring and crypto key are created
  • Verify backup bucket exists with correct naming
  • Confirm backup bucket has retention policy (30 days)
  • Verify versioning is enabled on backup bucket
  • Confirm uniform bucket-level access is enabled
  • Verify access logs bucket is created
  • Confirm Pub/Sub topic and subscription are created
  • Verify transfer job is created with correct schedule
  • Confirm log sink and log metric are deployed
Functional Validation
  • Test AWS IAM Role cross-cloud authentication
  • Run first manual transfer to validate end-to-end flow
  • Verify objects appear in GCS bucket with KMS encryption
  • Check Cloud Logging for transfer operation logs
  • Verify Pub/Sub notification on successful transfer
  • Confirm objects are in correct path ({env}/aurora-backups/)
  • Test monitoring alert by triggering a failed transfer
  • Verify NEARLINE transition rule is configured
  • Enable SIEM monitoring in Google SecOps
  • Set bucket_lock_enabled = true for production WORM mode
